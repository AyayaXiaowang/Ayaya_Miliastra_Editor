# assets/资源库/项目存档/测试项目/节点图/server/实体节点图/回归/端口类型

## 目录用途

存放 `测试项目` 下“端口类型/事件(event) 节点映射”相关的最小回归节点图（server/实体节点图）。

## 当前状态

- `回归_端口类型_event映射_信号_开始游戏.py`：监听信号 `开始游戏`（来自本项目存档的信号定义），用于覆盖 `node_def_ref.kind="event"` 的 `category/title -> builtin_key` 映射与离线迁移扫描（`scan-event-migration`）。
- `回归_端口类型_event映射_实体创建时.py`：覆盖 `node_def_ref.kind="event"` 的 NodeDef 定位口径（category/title -> builtin_key），用于保证事件节点映射/端口类型链路不回退。
- `回归_端口类型_别名字典KV_以键查询.py`：覆盖 typed dict alias 驱动的 K/V 类型收敛（`以键查询字典值` 的输出应可收敛为 `整数`），用于防止端口类型推断链路退化为泛型。

## 注意事项

- 本目录样本应只依赖 **本项目存档** 的管理配置（如 `管理配置/信号/`），避免引入其他包的信号/结构体。
- 文件名不要以下划线开头（否则 `validate-graphs --all` 默认收集会跳过）。
