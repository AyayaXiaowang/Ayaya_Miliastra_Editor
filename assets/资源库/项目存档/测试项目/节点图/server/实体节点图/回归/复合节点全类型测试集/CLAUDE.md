## 目录用途
复合节点 `.gil` 写回“全类型测试集”节点图（Graph Code）目录：多张宿主图覆盖复合节点 pins、信号、嵌套复合、多图复用等场景。

## 当前状态
- `TS_测试集_01_全类型_复合调用.py`：调用“全类型写入”复合节点，覆盖标量/列表/字典 typed alias 等类型写入；向量入参使用 `(x, y, z)`（括号 tuple）表达三维向量常量语义。
- `TS_测试集_02_复合内发送信号.py`：复合内发送信号 + 宿主图监听该信号（事件名=信号名）。
- `TS_测试集_03_嵌套复合.py`：宿主图调用“嵌套复合”，覆盖复合内调用复合 + 复合内发信号组合链路。
- `TS_测试集_04_复用复合_A.py` / `TS_测试集_05_复用复合_B.py`：两张图复用同一复合节点，覆盖多图共享 composite 定义。
- `TS_测试集_06_结构体与字典变量.py`：覆盖 GraphVariable 层的结构体/结构体列表/自定义变量快照/typed dict alias 的写回落盘；并显式为“获取节点图变量”的泛型输出添加中文类型注解，确保端口类型可实例化。

## 注意事项
- 遵循“避嫌”：graph_id/graph_name/composite_id/信号名均为测试集专用命名。
- 修改后建议跑：`python -X utf8 -m app.cli.graph_tools validate-file <节点图路径>`。
- 当复合节点端口期望 `配置ID列表/元件ID列表/阵营列表` 等强类型列表时，列表元素不要直接写裸数字常量；应先落到带中文类型注解的变量（如 `占位_配置ID: "配置ID" = 1`）再拼装列表，避免元素类型被推断为『整数』导致校验失败。
