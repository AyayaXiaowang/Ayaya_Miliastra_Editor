## 目录用途
存放 `测试项目` 的回归节点图（Graph Code）：用于 `.gil` 写回、导出与静态校验的稳定复现与验收。

## 当前状态
- 子目录按回归主题拆分（如复合节点全类型测试集、最小复现、写回/导出口径回归等）。
- `最小复现_create_prefab/`：创建元件（Create_Prefab）最小链路回归（端口索引一致性/写回/导入口径）。
- `写回回归_bool_pins/`：布尔值常量/端口映射写回回归（覆盖多类带 Bool 入参的执行节点，含自定义变量/节点图变量等）。
- `端口类型/`：端口类型与 `node_def_ref.kind="event"` 映射相关的最小回归节点图样本（供 `validate-graphs` 与离线迁移扫描复现）。
- `布局回归_列表迭代与分支_压力图/`：用于回归“列表迭代循环 + 双分支/多分支”组合场景下的分块编号与排版确定性。
- `最小复现_节点图挂载/`：节点图 `mount:` metadata 写回 root5 挂载字段的最小回归样本（验证挂载随导出产物写盘）。

## 注意事项
- 该目录只用于回归与诊断；不作为业务项目维护。
- 修改后建议跑：`python -X utf8 -m app.cli.graph_tools validate-file <节点图路径>`（单文件）或按需批量校验。
