## 目录用途
存放 `测试项目` 下的复合节点源码（类格式 `@composite_class`），用于覆盖 `.gil` 写回链路的复合节点落盘场景（section10 的 NodeInterface/CompositeGraph/port_mappings）。

## 当前状态
- 目录内复合节点均为 **server** 作用域。
- 复合节点覆盖：
  - 全类型虚拟引脚（含列表/字典 typed alias/结构体等）
  - 复合内发送信号（多参数、多类型）
  - 合并引脚（单外部 pin 扇出到多个内部端口）
  - 嵌套复合（复合内调用复合）
  - `TS_合并引脚_扇出_v1`：纯数据复合（仅数据入/数据出，不声明流程口）；用于验证同一外部 pin 映射到多个内部端口（mapped_ports 多条）的写回稳定性。
  - `TS_嵌套复合_组合_v1`：包含流程口；外部入参先经 `获取局部变量(...)` 生成稳定数据来源，再驱动嵌套复合调用（避免 mapped_ports=0/缺数据来源）。
  - 列表/字典类复合节点的字典端口使用 **别名字典类型**（例如 `字符串-整数字典`），以提供稳定的 K/V VarType 证据供 `.gil` 写回落盘。
  - 列表/字典类复合节点会对“接口上声明但子图中未使用”的入参做显式节点消费（如 `获取列表长度(...)`），避免虚拟引脚因 `mapped_ports=0` 在 UI/写回/导出侧被视为“未使用”而不渲染。
  - `TS_最小布尔复合_v1`：最小复现复合节点，仅 1 组布尔数据入/数据出虚拟引脚（不声明流程口）；用于验证复合子图 inner_nodes 的 Bool 类型载体 pin（NodePin/ConcreteBase/EnumBaseValue.field_106）写回口径。

## 注意事项
- 不写 try/except；校验失败直接抛错（fail-fast）。
- 复合节点间 import 使用真实资源路径：`from 资源库.项目存档.测试项目.复合节点库... import ...`。
- 对返回“泛型”输出的节点（如字典查询类节点）：必须对接收变量做显式中文类型注解（例如 `x: "整数" = ...`），以保证复合节点源码校验能静态确定端口类型。
