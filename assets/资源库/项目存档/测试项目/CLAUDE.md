## 目录用途
默认未归属项目存档（package_id=`测试项目`）：用于承载测试/回归/诊断类资源与工具链夹具（复合节点库 + 节点图 Graph Code 等），便于在不影响业务项目存档的情况下做稳定验证。

## 当前状态
- package_id 固定为 `测试项目`（见 `engine.utils.resource_library_layout.DEFAULT_UNCLASSIFIED_PACKAGE_ID`）。
- 当前包含“复合节点 `.gil` 写回全类型测试集”：
  - `复合节点库/`：测试用复合节点（复合内发送信号、合并引脚、嵌套复合、全类型 pins）
  - `节点图/server/实体节点图/回归/复合节点全类型测试集/`：多张宿主图（含多图复用同一复合）
  - `管理配置/信号/`：测试集专用信号定义（供发送/监听信号节点静态补全参数端口与类型）

## 注意事项
- 遵循“避嫌”原则：该目录下资源命名与外部参考 `.gil` 不一致（全新 graph_id/composite_id/信号名）。
- 本目录只用于回归与诊断；不作为业务项目维护。
