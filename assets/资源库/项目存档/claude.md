# assets/资源库/项目存档

## 目录用途

`assets/资源库/项目存档/` 是项目存档资源根：每个子目录对应一个项目存档（package），用于承载该存档独有资源（节点图、复合节点库、管理配置、战斗预设等）。与 `assets/资源库/共享/` 共同构成资源库的两大资源根。

## 当前状态

- `示例项目_第七关/`：示例项目存档（用于对外/回归演示；资源范围以该目录为边界）。
- `示例项目模板/`：示例项目模板存档（用于复制创建新项目；提供模板示例资源）。
- `测试项目/`：回归/诊断夹具落点（不作为业务项目维护）。

## 注意事项

- 对外仓库仅版本化 `示例项目_第七关/`、`示例项目模板/` 与 `测试项目/`；其他子目录视为本地/需授权内容，默认不提交。
- 临时/诊断/回归用项目存档优先放入 `测试项目/`；如需单独隔离 package，可使用 `_tmp_` 前缀（历史命名 `__tmp_*`、`verify_*`、`cmp_*` 等应归并到 `_tmp_*` 语义下）。
- 不要在 `assets/资源库/` 的 legacy 根直接新增资源文件；请放入某个 `项目存档/<package>/` 或 `共享/` 资源根下。
- 资源归属以“目录即存档”为准：资源文件位于哪个 `项目存档/<package>/` 目录，就属于哪个项目存档。
