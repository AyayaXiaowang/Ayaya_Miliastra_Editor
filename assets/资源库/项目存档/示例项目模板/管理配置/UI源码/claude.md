## 目录用途
- 存放项目存档内的 **UI 源码**（HTML + CSS）：供“千星沙箱网页处理工具（Web-first UI 转换器）”读取并派生运行时缓存产物（不落资源库）。
- 约定：**一个 HTML 文件对应一个界面布局**（一个网页 = 一个 UILayout）。

## 当前状态
- 作为目录骨架存在，供新建项目存档复制后直接放置 UI 源码文件。
- 保存后若启用私有扩展转换器：会自动将 HTML 派生物写入 `app/runtime/cache/ui_artifacts/<package_id>/`（例如 HTML bundle、ui_key→guid 注册表、UI 交互映射）。

## 注意事项
- 这里的 `*.html` 不会被资源索引当作运行时资源；HTML 是 **唯一真源**，派生 JSON 属于运行时缓存，不参与版本管理。
- 动态文本占位符写法（推荐显式声明来源）：
  - `{{lv.<变量名>}}`：关卡实体自定义变量（来自当前项目引用的关卡变量文件或共享变量文件）
  - `{{ps.<变量名>}}`：玩家模板自定义变量（仅允许玩家模板 `metadata.custom_variable_file`；禁止局内存档变量）
- 禁止使用 `{{<变量名>}}` 这类无前缀写法（避免来源不明导致导出后缺变量才暴露）。
- UI 多状态写法（推荐）：在“组根节点”写 `data-ui-state-group`，在子状态节点写 `data-ui-state` / `data-ui-state-default`；**状态节点禁止 `display:none`**（否则无盒子无法扁平化/导出），初始隐藏请用“非默认态隐藏”策略并保持盒子存在。
- 页面不维护 `data-ui-variable-defaults`；变量默认结构以 `管理配置/关卡变量/自定义变量注册表.py` 为准（由工具链按需注入/同步）。
- 本目录只放源码与素材引用（相对路径）；不要把导出的 JSON、GIL 或运行时缓存写到这里。
