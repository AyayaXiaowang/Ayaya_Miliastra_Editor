## 目录用途
- 存放服务器端节点图的回归/边界覆盖用例（Graph Code 脚本）。
- 重点覆盖：复合节点实例化、复合节点多流程出口、复合节点多数据引脚、复合节点嵌套、复合节点内含信号发送等导出/解析路径。

## 当前状态
- 本目录脚本均应保持 **严格校验通过**，并提供 `__main__` 的脚本级自检入口（`validate_file_cli`）。
- 本目录脚本以“测试覆盖”为目标，允许比 `模板示例/` 更密集地构造组合场景，但仍保持可读与最小化。

## 注意事项
- 不在图逻辑里写“判空/数据是否存在”类防御代码；缺数据应直接报错暴露问题。
- 若新增/修改脚本，必须先跑验证：`python -X utf8 -m app.cli.graph_tools validate-file <文件路径>`。
