## 目录用途
- 存放“示例项目模板”专用的复合节点源码（类格式 `@composite_class`）。
- 作为节点图示例（`节点图/server/实体节点图/`）的可复用子图/子能力，便于演示复合节点的调用、引脚设计与嵌套组合。

## 当前状态
- 目录内复合节点均为 **server** 作用域。
- 复合节点采用“流程入口 + 虚拟引脚声明（流程入/流程出/数据入/数据出）”的类格式，供导出 `.gia`/写回链路识别与打包。
- 复合节点示例覆盖多样性场景：空复合（仅流程穿透）、单数据引脚（最小 data in/out）、合并引脚（同一外部输入映射到多个内部节点端口）、多引脚模板、多分支、复合内发送信号、以及少量“嵌套复合”用于边界验证。

## 注意事项
- 不写 try/except：错误直接抛出，便于定位复合节点契约问题。
- 复合节点文件只需保证能被 Python AST 解析；运行期解析/校验由统一验证入口处理。
- 若 `@flow_entry` 方法声明了 `流程入/流程出`，方法体内必须包含至少一个可建模的流程节点（执行节点或流程控制结构），否则虚拟流程引脚无法映射到子图流程连线，导出 `.gia` 也会失真。
- 若复合节点在子图中调用其它复合节点，实例属性名建议包含目标复合节点类名/节点名，保证解析器可稳定识别为“复合节点实例调用”。
- 复合节点之间的相互导入应使用“真实资源路径”以支持直接运行自检：示例项目模板下写为 `from 资源库.项目存档.示例项目模板.复合节点库.composite_xxx import Xxx`；不要使用不存在的 `资源库.复合节点库` 快捷路径。
