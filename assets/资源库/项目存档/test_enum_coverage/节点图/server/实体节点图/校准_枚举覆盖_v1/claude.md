## 目录用途
- 存放 **自动生成** 的 server 侧“枚举候选项覆盖图”（Graph Code，Python 类结构）。
- 目标：覆盖所有 server 节点中声明了 `input_enum_options/output_enum_options` 的枚举候选项；按事件/主题拆分为多张小图，便于阅读与维护。

## 当前状态
- 目录内包含多张覆盖图（`*.py`），命名以 `校准_枚举覆盖_v1_server...` 为前缀；每个文件通常只包含 1 个事件入口，单事件节点数控制在 20 以内。
- 保留 `校准_枚举覆盖_v1_server_level.py`：关卡挂载专用（仅放置事件节点【实体销毁时】以避免挂载冲突）。

## 注意事项
- 校验入口：运行本项目的 `节点图/校验节点图.py`（会扫描本目录所有节点图文件并调用引擎校验）。
- 本目录节点图仅用于静态解析/校验与枚举覆盖率回归，不依赖真实配置或运行期对象。
- `GUID/配置ID/元件ID` 等占位常量遵循 “1~10 位纯数字（支持整数或数字字符串）” 约束，避免触发静态校验错误。
