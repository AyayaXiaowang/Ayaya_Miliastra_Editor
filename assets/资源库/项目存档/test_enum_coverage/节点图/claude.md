## 目录用途
- 存放本测试项目的节点图 `.py` 代码文件（Graph Code）。
- 目标：用一套“枚举覆盖图”确保所有声明了枚举候选项的节点，其每个枚举值都在图中至少出现一次。

## 当前状态
- `server/实体节点图/校准_枚举覆盖_v1/`：server 枚举覆盖图（类结构 Graph Code）。
- `client/技能节点图/校准_枚举覆盖_v1/`：client 枚举覆盖图（技能节点图，入口为 `on_节点图开始`）。
- `校验节点图.py`：项目级节点图校验入口（资源库专用）。

## 注意事项
- Graph Code 仅用于静态解析与校验：禁止写判空/“数据是否存在”分支等无意义逻辑。
- 若节点枚举候选项（`input_enum_options/output_enum_options`）发生变化，应同步更新覆盖图，并通过本目录的校验入口与 pytest 覆盖测试。
