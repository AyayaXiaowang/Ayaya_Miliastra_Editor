## 目录用途
`src/workbench/` 提供 Workbench 左侧“UI源码/变量”等浏览器能力：从主程序 `/api/ui_converter/*` 拉取清单与内容，并把选中文件加载到编辑器区域供渲染/扁平化/导出复用。

## 当前状态
- `ui_sources.js`：UI源码文件列表（项目 + 共享），支持搜索、点击打开、保存到项目目录。
  - 打开文件时会**立即更新 currentSelection（先于 fetch 返回）**，并触发上层 `onSelectionChanging` 回调，用于让 browse 流水线/重操作尽快感知“选择已变化”并停止，避免旧任务继续写入输出。
  - 支持“仅显示已扁平化”过滤：由上层在 browse 模式开启，用于在自动预扁平化完成后再显示列表。
  - 支持 `onCatalogRefreshed(items)` 回调：上层可在刷新清单后启动批量预扁平化流程，并决定是否自行渲染列表。
- `variables.js`：变量目录浏览与一键插入占位符。

## 注意事项
- 浏览器式工作流（browse mode）：打开 UI源码文件时不会强制切换到“编辑”Tab；由上层（`workbench_main/index.js`）决定是否自动执行“校验→扁平化→导出 bundle”并默认预览扁平化结果。
- `ui_sources.js` 提供“一键处理并导入（项目 UI源码）”批处理入口：按清单依次打开每个项目 HTML，由上层 browse 流程自动完成扁平化/导出/导入（不依赖“Browse 自动导入”勾选状态）；用于快速把一批未处理页面同步到当前项目存档。
- 该目录只负责“读取/保存/列表渲染”，不承担扁平化与导出逻辑；导出由 `workbench_main/flattening.js` 负责。
- 该目录代码运行在浏览器侧：禁止引入 Node.js API。
- 模块应避免自行扫描/修改页面结构；DOM 引用统一来自 `src/dom_refs.js`（ID 强绑定）。
