# 用户必读：只写节点图（Graph Code）

本仓库包含“节点图代码解释器 / 校验器 / UI 查看器 / 自动化搭图工具”。  
对绝大多数使用者来说，你只需要**编写节点图 Graph Code**，不需要也不应该修改解释器源码。

## 你应该改哪些文件

- **节点图（Graph Code）**：
  - `assets/资源库/共享/节点图/<server|client>/**/*.py`
  - `assets/资源库/项目存档/<项目存档名>/节点图/<server|client>/**/*.py`
- **（可选）复合节点**：
  - `assets/资源库/共享/复合节点库/**/*.py`
  - `assets/资源库/项目存档/<项目存档名>/复合节点库/**/*.py`
- **（可选）资源与配置**：
  - `assets/资源库/共享/管理配置/**`、`assets/资源库/共享/实体摆放/**` 等
  - `assets/资源库/项目存档/<项目存档名>/管理配置/**`、`assets/资源库/项目存档/<项目存档名>/实体摆放/**` 等

## 你不应该改哪些文件（解释器/工具链）

以下目录默认视为只读，由项目作者维护：

- `engine/`：Graph Code 解析 / 建模 / 校验 / 排版等核心引擎
- `app/`：UI、运行态、自动化装配
- `plugins/`：节点实现与静态注册表

即使你发现了解释器 BUG，也建议**反馈给作者**，而不是本地硬改后继续使用（容易导致后续升级/协作不可控）。

## 标准工作流（推荐）

### 1) 编写节点图源码

在任意“资源根目录”下创建/修改节点图 `.py` 文件（共享/项目存档都可以）：

- `assets/资源库/共享/节点图/<server|client>/...`
- `assets/资源库/项目存档/<项目存档名>/节点图/<server|client>/...`

### 2) 运行校验（必须）

- 校验单个文件：

```powershell
python -X utf8 -m app.cli.graph_tools validate-graphs -f assets\资源库\项目存档\你的项目存档\节点图\server\你的图.py
```

- 全量校验（所有节点图/复合节点）：

```powershell
python -X utf8 -m app.cli.graph_tools validate-graphs --all
```

校验不通过请按输出提示修复后再继续。

#### （推荐）项目存档级校验（目录模式）

当你修改了项目存档内的资源（模板/实体摆放/管理配置/UI 等），建议额外跑一次“项目存档级”校验，用来提前发现：
- 节点图挂载缺失（图文件存在但没挂到任何模板/实体摆放）
- 资源引用不闭合（UI布局引用了不存在的 UI控件模板、管理配置引用了不存在的变量文件等）
- GUID/资源 ID 冲突与不合法写法

校验全部项目存档：

```powershell
python -X utf8 -m app.cli.graph_tools validate-project
```

仅校验单个项目存档：

```powershell
python -X utf8 -m app.cli.graph_tools validate-project --package-id "你的项目存档名"
```

备注：旧命令名 `validate-package` 仅作为兼容保留，文档与后续推荐统一使用 `validate-project`。

### 3) 在 UI 中预览

```powershell
python -X utf8 -m app
```

### 4) （可选）自动搭图到千星沙箱

在 UI 的“任务清单”页选择对应节点图并执行。

## 发现问题如何反馈（推荐格式）

为了让作者能最快定位问题，请尽量提供：

- **报错输出全文**（包含堆栈）
- **对应节点图文件路径**（或最小可复现的单文件）
- **你的环境信息**：Windows 版本、分辨率/缩放、Python 版本
- **你期望的行为**与**实际发生的行为**

反馈渠道：

- QQ 群：`1073774505`
- GitHub：请提交 Issue（附上最小可复现）
