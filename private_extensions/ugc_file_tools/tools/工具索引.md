# ugc_file_tools 工具索引（可检索）

本文件由 `ugc_file_tools/tool_registry.py` 生成。
- 如需新增/改名/调整风险与说明：修改注册表后重新生成本文件。
- 生成/校验脚本：`python -X utf8 -m ugc_file_tools.tools.generate_tool_index_md --help`

本文件用于快速回答两个问题：
- **“我该跑哪个入口？”**
- **“这个入口会不会写盘/改存档？”**

> 约定：下文工具名均可用 `python -X utf8 -m ugc_file_tools tool <工具名> --help` 查看完整参数与示例。

## 统一运行姿势（推荐）
- **统一入口（最常用）**：`python -X utf8 -m ugc_file_tools --help`
  - 等价入口：`python ugc_file_tools/ugc_unified.py --help`
- **统一入口转发单工具**：`python -X utf8 -m ugc_file_tools tool <工具名> --help`
- 不再提供 `ugc_file_tools.tools.<模块名>` 单独入口（避免同名双入口）；统一用 `ugc_file_tools tool`。

PowerShell 提醒：不使用 `&&` 串联命令；需要多条命令请分行执行。

`.gia` 提醒：所有生成/写回 `.gia` 的工具都会将产物**最终导出**到固定目录：
- `C:\Users\xiaowang\AppData\LocalLow\miHoYo\原神\BeyondLocal\Beyond_Local_Export`
- `ugc_file_tools/out/` 仍会落盘作为中间产物与可追溯输出。

## 风险标识（读写与破坏性）
- **[只读]**：只读取/解析并输出报告，不改磁盘（或仅写到临时输出文件）
- **[写盘]**：会生成导出产物/报告/代码到 `out/` 等目录（通常可重复生成）
- **[危险写盘]**：会生成新的 `.gil`/对存档做写回类操作；务必先备份输入存档，并确认输出路径

## 推荐入口（统一 CLI）
- **`ugc_file_tools` [写盘/危险写盘]**：统一入口（UI / project / entity）。
  - `python -X utf8 -m ugc_file_tools --help`

## 解析导出（`.gil` → 项目存档/结构化数据）
- **`extract_gil_to_package` [写盘]**：.gil 导出为项目存档目录（核心实现：gil_package_exporter/）。
- **`parse_package` [只读/写盘]**：解析项目存档并输出结构化摘要 JSON。
- **`parse_gil_to_model` [写盘]**：一键入口：导出项目存档 → 输出摘要（可选 codegen）。
- **`parse_gil_to_full_model` [写盘]**：全量入口：导出 → 摘要 → codegen → 校验（更重）。
- **`gil_to_readable_json` [写盘]**：自研可读 JSON dump（用于分析 protobuf-like payload）。

## 节点图 IR（可读节点图导出/分析）
- **`export_graph_ir_from_package` [写盘]**：从项目存档导出 Graph IR（JSON/Markdown）。
- **`parse_gil_to_graph_ir` [写盘]**：从 .gil（可选先导出项目存档）导出 Graph IR。
- **`parse_gil_payload_to_graph_ir` [写盘]**：直接解析 .gil payload（10.1.1 NodeGraph blob）导出 Graph IR（更贴近 .gia pins/edges）。
- **`parse_gia_to_graph_ir` [写盘]**：从 .gia 解析并导出 Graph IR（结合 node_data/index.json 补全提示）。
- **`inspect_parsed_node_graphs` [只读]**：打印内部解析后的 nodes/edges 摘要，便于 sanity check。
- **`inspect_ui_guid` [只读]**：给定一个 UI guid，反查对应的 ui_key / UI record 名称 / 父子关系（只读）。

- **`gia_build_decorations_bundle` [写盘]**：从 decorations_*.report.json 生成“装饰物挂件类” .gia（基于 base .gia 作为结构模板）。
