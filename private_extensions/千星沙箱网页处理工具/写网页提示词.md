一、 核心技术限制
技术栈：仅限 HTML + CSS。
你要写的是一个纯静态展示用的网页，没有任何交互。所有的按钮都只能看，不管用户如何要求，你都是写纯静态的。

❌ 禁止 JS（除非用户明确要求用于本地预览的极简脚本，且绝对不能影响无 JS 环境下的静态导出）。
文件结构：单文件 HTML。
分辨率适配：必须支持以下 4 种尺寸，且无滚动条（使用 `--ui-scale` 和 `clamp` 等自适应容器大小）：

1920 × 1080
1600 × 900
1560 × 720
1280 × 720

**字号硬约束**：文本字号（`font-size`）必须是固定的 `px` 值（禁止 `vw/vh/%/em/rem/clamp()` 或媒体查询分支改变字号），否则导出工具会报错并阻断导出。

补充：文件落点与全局声明（必须遵守）
- HTML 源码保存位置：`assets/资源库/项目存档/<项目>/管理配置/UI源码/`
  - 约定：一个 HTML 文件对应一个界面布局（一个网页 = 一个 UILayout）
- **预览初始态覆盖（可选）**：可在容器上声明 `data-ui-preview-initial-states="<group>=<state>"`，用于在 Web 预览时自动应用状态（不影响导出）。

二、 UI 控件导出标注与动态文本（关键更新）

所有需要导出到游戏内的控件，必须使用 `data-ui-*` 属性进行标注。

1. 控件基础标注
- `data-ui-key="<唯一ID>"`：必填，导出控件的唯一标识。
- `data-ui-role="<类型>"`：必填，可选值包括 `label`（文本）、`button`（按钮）、`progressbar`（进度条）、`panel`（面板/容器）。

2. 动态文本占位符（必须写在 data-ui-text 属性中）
- 动态文本必须通过 `data-ui-text` 属性声明，**不要**直接写在 HTML 的 `innerText` 里（`innerText` 仅作为网页预览的静态占位）。
- 占位符格式为 `{1:lv.<变量名>}` 或 `{1:ps.<变量名>}`：
  - `lv.<变量名>`：关卡实体自定义变量。
  - `ps.<变量名>`：玩家模板自定义变量。
- 示例：
  ```html
  <div data-ui-key="header_title" data-ui-role="label" data-ui-text="{1:lv.UI结算_文本.标题}">这里是预览文本</div>
  ```
- 文本对齐：建议显式声明 `data-ui-text-align="left|center|right"` 和 `data-ui-text-valign="top|middle|bottom"`，避免默认推断导致对齐漂移。

3. 进度条绑定
- 必须包含 `data-progress-shape="横向"`（或纵向）。
- 变量绑定直接写在属性里，不需要 `{1:}` 前缀，且**必须是标量变量**（不能含 `.`）：
  ```html
  <div data-ui-key="stat_01_progressbar" data-ui-role="progressbar"
       data-progress-shape="横向"
       data-progress-current-var="ps.ui_battle_integrity"
       data-progress-min-var="ps.ui_battle_i_min"
       data-progress-max-var="ps.ui_battle_i_max"></div>
  ```

三、 视觉与图层构成规范（Tone-3 风格）

本项目统一使用 `.tone-3` 结构来实现按钮和面板的立体感（底色 + 底部短唇边压暗）。

1. Tone-3 基础结构
```html
<div class="tone-3">
    <span class="tone-3-stripe tone-3-stripe--top" aria-hidden="true"></span>
    <span class="tone-3-stripe tone-3-stripe--bottom" aria-hidden="true"></span>
    <span class="tone-3-stripe tone-3-stripe--deep" aria-hidden="true"></span>
    <!-- 实际内容放在这里 -->
    <span class="content">内容</span>
</div>
```

2. Tone-3 CSS 规范
```css
.tone-3 {
    position: relative;
    border-radius: var(--radius-md);
    overflow: hidden;
    background-color: var(--tone-top, var(--tone-mid));
    box-shadow: none; /* 全局取消左上角高光 */
}
.tone-3-stripe { position: absolute; left: 0; right: 0; z-index: 0; pointer-events: none; }
/* 顶部高光：统一关闭，避免额外拆出控件。注意：必须用 visibility: hidden，禁止 display: none */
.tone-3-stripe--top { background-color: transparent !important; visibility: hidden !important; }
/* 底部短唇边：保留 */
.tone-3-stripe--bottom {
    top: auto; bottom: 0; height: 6px;
    background-color: var(--shadow-mask-25);
}
/* 底部二次加深：统一关闭 */
.tone-3-stripe--deep {
    bottom: 0; height: 0;
    background-color: transparent !important; visibility: hidden !important;
}
.tone-3 > :not(.tone-3-stripe) { position: relative; z-index: 1; }
```

3. 阴影 (Drop Shadow)
- **CSS 实现**：使用 `box-shadow`。
- **强制参数**：必须写成带 alpha 的颜色（如 45% 透明度），禁止纯黑。
  - ✅ 推荐：`rgba(14, 14, 14, 0.45)` 或 `#0E0E0E73`
  - ❌ 禁止：`#000` / `#000000` / `#0E0E0E`

四、 布局与层级堆叠 (Z-Index)
为了配合工具的扁平化覆盖逻辑，CSS 的层级管理必须清晰：

自然堆叠顺序：遵循 DOM 树顺序。
内部层级逻辑：对于同一个 HTML 元素，视觉上的覆盖关系永远是：文本 > 主体 > 边框 > 阴影。
扁平化层级：扁平化导出时不读取 CSS 的 `z-index`，如需强制层级，请在元素上使用 `data-flat-z-bias="1000000"`。

五、 颜色板 (严禁例外)
背景/边框/阴影颜色只能来自以下集合（严禁例外）；**文本颜色允许任意**：

5.1 基础色 (Base)
建议使用 CSS 变量：
`--c-green: #92CD21;`
`--c-white: #FFFAF0;`
`--c-yellow: #FFD700;`
`--c-blue: #36F3F3;`
`--c-red: #FF6B6B;`

5.2 阴影色 / 黑色替代
- 45% 阴影盖色（墨色）：`--c-ink: #0e0e0e73;`
- 25% 阴影盖色（浅墨色）：`--c-ink-soft: #0e0e0e40;`
- ❌ 禁止在非文本颜色中使用不透明近黑（例如 `#0E0E0E` / `#000` / `#000000`），请统一使用 `--c-ink` 作为边框或背景的“黑色”。

六、 游戏区域（挖空 / 漏出游戏内容）
- 用途：声明“这里是游戏画面露出来的区域”，背景层在扁平化/导出时会自动把该区域挖空。
- 写法：`class="game-cutout"`
- 约束：必须是透明（`background-color: transparent`），不允许边框/阴影/文字。

七、 多状态控件（按钮/面板的 enabled/disabled/选中/未选中 等）
- 目标：在“纯静态 HTML+CSS”里表达“同一个控件有多种状态”，初始只显示默认态；运行时由节点图负责切换显隐。
- 写法（强约束）：
  - `data-ui-state-group="<组名>"`：必填，同组的不同状态必须一致。
  - `data-ui-state="<状态名>"`：必填，如 `enabled` / `disabled`。
  - `data-ui-state-default="1"`：仅默认态填写（同组只允许一个默认态）。
- **绝对禁止 `display:none` 隐藏非默认态**：`display:none` 会导致元素没有盒子，Workbench 无法提取坐标与导出。
- 隐藏非默认态必须用：`visibility:hidden` 和 `pointer-events:none`。

推荐的多状态按钮布局（槽位叠放法）：
```html
<div class="action-btn-cell" style="position: relative; width: 200px; height: 60px;">
    <!-- Enabled 态 -->
    <button class="btn tone-3" type="button"
            data-ui-key="btn_allow" data-ui-role="button"
            data-ui-state-group="btn_allow_state" data-ui-state="enabled" data-ui-state-default="1">
        <span class="btn-text">允许进入</span>
    </button>
    <!-- Disabled 态 -->
    <button class="btn tone-3 disabled" type="button"
            data-ui-key="btn_allow" data-ui-role="button"
            data-ui-state-group="btn_allow_state" data-ui-state="disabled">
        <span class="btn-text">允许进入</span>
    </button>
</div>
```
对应的 CSS：
```css
.action-btn-cell .btn[data-ui-state-group] {
    position: absolute; inset: 0;
    visibility: hidden; pointer-events: none;
}
.action-btn-cell .btn[data-ui-state-group][data-ui-state-default="1"] {
    visibility: visible; pointer-events: auto;
}
```

八、 可交互按钮（强制规范）
- 目标：让 Workbench **稳定**把“可点击控件”导出为 **道具展示（可交互锚点）**。
- 优先使用 `<button type="button">` 来写任何“看起来是按钮/需要点击”的控件。
- 必须显式标注其语义为按钮：`data-ui-role="button"`。
- 装饰性按钮：若某个按钮仅用于视觉占位，不希望导出为可交互按钮锚点，在该元素上标注 `data-ui-export-as="decor"`。

九、 UI 交互动作标注（推荐，用于节点图逻辑解耦）
- 写法（加在可点击按钮根元素上）：
  - `data-ui-action="<动作ID>"`：例如 `ui.select_level` / `ui.exit`
  - `data-ui-action-args="<参数文本>"`：参数原样透传

十、 UI 交互按键码标注（可选；强约束：同页≤14且不重复）
- 游戏内“可交互道具展示”需要绑定“奇匠按键码”。同一个页面内：
  - 可交互按钮数量 **不能超过 14**
  - 每个按钮必须占用 **不同的按键码（1..14）**
- 写法：`data-ui-interact-key="1"` ~ `"14"`

十一、 按钮“装备配置ID变量”（自动填充；无需真实道具存在）
- 默认值：若不写任何标注，工具会自动绑定到：`玩家自身.UI_交互按钮_装备配置ID`。
