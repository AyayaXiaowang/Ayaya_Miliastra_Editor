# private_extensions 目录说明

## 目录用途
- 存放扩展与工具链：用于研发期实验、导入/写回验证、以及与真源（官方编辑器/真实存档）的桥接。
- 扩展能力通常不作为主线公共 API 的强依赖；若某个能力需要长期维护并作为核心能力提供，优先迁移到 `engine/`、`app/`、`plugins/` 或 `tools/`。

## 当前状态
- `ugc_file_tools/`：`.gil/.gia` 解析/导出/对照诊断/写回工具链（统一产物目录：`ugc_file_tools/out/`）。
- `千星沙箱网页处理工具/`：Web-first 的 UI 预览/导出/导入工具 + 主程序私有插件入口。
- `shape-editor/`：网页画板/像素工作台 + 导出 `.gia` 的内置扩展与后端桥接层（随仓库分发）。
- `third_party/`：第三方参考/对照数据（不作为插件加载目标；可随时清理重建）。

## 注意事项
- 扩展的加载/入口约定以 `app.common.private_extension_loader` 的实现为准（通常为“每个插件目录一个 `plugin.py`”）。
- 遵循 fail-fast：不吞错；并避免在模块顶层导入重型依赖（尤其是 PyQt6）。
- 目录名 `private_extensions/` 为历史约定保留：用于“扩展加载机制”的落点，并不意味着目录内工具仍为私有。
- 子目录应提供简洁的 `claude.md`，仅描述“目录用途/当前状态/注意事项”，不写修改历史。
