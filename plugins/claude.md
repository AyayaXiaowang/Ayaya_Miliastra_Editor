# 目录用途
可插拔实现层：节点实现与与环境耦合的扩展（如自动化适配）。

# 公共 API
无（以源码文件形式提供节点实现；节点库由 `engine.nodes.pipeline` 静态解析 `@node_spec` 构建，不导入实现模块）。

# 依赖边界
- 允许依赖：`engine/*`
- 禁止依赖：`app/*`（除装配）、`assets/*`（除只读数据）

# 当前状态
- 节点实现已按 `server` / `client` / `shared` 分域存放；节点库构建由 `engine.nodes.pipeline` 扫描解析 `plugins/nodes/**.py`（**只解析不导入**），不再维护 `plugins/nodes/registry.py` 一类的静态注册表文件。
- 新增或调整节点实现后，无需生成注册表；建议跑一次节点图校验与 pytest 回归，确保节点库契约与端口声明不回归。

# 注意事项
- 节点库发现采用“V2 AST 管线”：只解析 `@node_spec`，不导入实现模块，避免副作用；不要在模块顶层写 I/O、环境探测等副作用代码。 
- **默认禁止**在本目录随意新建节点实现/节点文件：除非用户明确指定“在 `plugins/`（或 `plugins/nodes/`）创建节点/实现插件节点”，否则不要在这里新增节点。
- **严禁为了通过校验而补节点**：节点图校验报 `CODE_UNKNOWN_NODE_CALL` / “节点不在当前作用域节点库中”，说明该 scope 下不能用这个节点；只能改节点图/改写法/换已有节点，不能通过新增 `plugins/nodes/**` 来“让校验通过”。
- 用户要求“新建/修改节点图（Graph）”时，默认目录是资源库任意资源根目录下的 `节点图/`（按 `server/` / `client/` 归档），而不是 `plugins/`：
  - `assets/资源库/项目存档/<项目存档名>/节点图/<server|client>/...`
  - `assets/资源库/共享/节点图/<server|client>/...`
- 用户只说“新增节点/补节点”但未指明位置时：先确认是“节点图”还是“节点实现”；节点图去上述资源库目录，节点实现需先向用户确认是否应放在插件层（本目录）还是引擎核心目录。
- 仅当需求涉及“与环境耦合的扩展能力”（如自动化适配、外部依赖隔离）或用户明确点名插件层时，才考虑在本目录新增节点实现。
- **本目录 `claude.md` 为公开文档**：不要在这里写任何私有/绝密功能细节。若需要本地私有扩展，请放在工作区根目录的 `private_extensions/`（默认 `.gitignore` 忽略），并在该目录下维护仅本机可见的 `claude.md`。
