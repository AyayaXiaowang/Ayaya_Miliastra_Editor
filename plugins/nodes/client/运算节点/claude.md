# Client 运算节点目录

## 目录用途
- 客户端运算/转换节点集合：数学函数、向量、逻辑判断、列表处理等。

## 当前状态
- 节点库构建由 V2 AST 管线扫描 `@node_spec`（只解析不导入）；运行期需要执行节点函数时，由 `app.runtime.engine.node_impl_loader` 按需加载并注入。
- 共用算法可沉淀在 `plugins.nodes.shared.client_运算节点_impl_helpers`。
- 支持参数面板额外配置（如数据类型转换的目标类型），但端口依旧保持最简输入/输出形态。
- 基础算术节点（加/减/乘/除）端口定义已对齐语法糖与校验约定：`左值/右值 -> 结果`，并声明输入/输出泛型约束，避免解析后出现端口缺失导致的连线渲染异常。

## 注意事项
- 保持节点纯粹，不产生副作用，便于在客户端流程中复用。
- 统一日志出口为 `engine.utils.logging.logger`。
- 数值/位运算节点、比较类运算节点和数据类型转换节点需在 `@node_spec` 中声明泛型约束；目录内的基础算术节点（加减乘除等）已限定仅接受整数或浮点并输出同类类型，数据类型转换节点则限制可接受/输出的基础数据类型集合，以便编辑器与校验器提前发现错连。
- 【数据类型转换】节点的“可转换对”以 `engine.type_registry.TYPE_CONVERSIONS` 为权威；泛型约束只声明输入/输出集合，输入→输出的组合合法性由编辑器与校验器按转换表联动检查。
- 比较类运算节点端口命名需与语法糖转换保持一致：`是否相等` 使用 `输入1` / `输入2`；数值比较（`是否大于/是否小于/是否大于等于/是否小于等于`）使用 `左值` / `右值`。
