# Client 其他节点目录

## 目录用途
- 存放客户端“其他类型”节点，目前用于承载技能节点图的起点等特殊入口类功能。

## 当前状态
- 节点库构建由 V2 AST 管线扫描 `@node_spec`（只解析不导入）；运行期需要执行节点函数时，由 `app.runtime.engine.node_impl_loader` 按需加载并注入。
- 提供技能节点图的统一入口节点：`节点图开始`，仅提供一个流程出口，不做任何副作用，用于在技能节点图中作为逻辑的起点。

## 注意事项
- 未来如需扩展更多“其他”节点，请保持文件命名与节点名称一致。
- 依赖边界与其余客户端节点一致：仅依赖 `engine` 与 `plugins.nodes.shared`。
