# Client 执行节点目录

## 目录用途
- 客户端执行节点集合，涵盖镜头、输入、局部状态修改等可在客户端完成的动作。
- 每个文件实现一个 `@node_spec` 节点函数。

## 当前状态
- 节点库构建由 V2 AST 管线扫描 `@node_spec`（只解析不导入）；运行期需要执行节点函数时，由 `app.runtime.engine.node_impl_loader` 按需加载并注入。
- 共享逻辑位于 `plugins.nodes.shared.client_执行节点_impl_helpers`（如有）。

## 注意事项
- 不访问服务器专属 API，必要的 RPC 交给节点自身或 shared helper。
- 日志输出统一走 `engine.utils.logging.logger`。
- 若节点需要界面交互，保证只调用允许的 UI 抽象接口。
- 若输入端口类型声明为“枚举”，应在 `@node_spec` 中补齐 `input_enum_options`（例如目标筛选、触发类型、攻击类型/打击类型、受击表现等级等），以支持编辑器下拉选择与校验层对枚举字面量做严格约束。
- 攻击盒相关节点存在多类参数：枚举端口（攻击盒类型/攻击层筛选/扇形检测方向/受击击退朝向等）必须补齐候选项；数值/向量端口（如长方体缩放、球体半径）保持为对应的数据类型，避免把数值端口误标为枚举导致节点图用字符串占位。
