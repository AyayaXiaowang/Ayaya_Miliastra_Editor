# 目录用途
- 复合节点（Composite）相关的图解析与格式转换工具：负责将复合节点的类格式解析为统一的 IR，并跟踪参数使用情况。
- `pin_api.py` 维护 `流程入/流程出/数据入/数据出` 等声明式辅助函数，运行期 no-op，方便节点图代码引用。
- `pin_marker_collector.py` 负责扫描 AST，提取上述辅助函数与方法签名中蕴含的自动引脚声明。

# 当前状态
- `source_format.py` 作为复合节点源码格式（payload / @composite_class）的单一事实来源：统一提供 payload 提取与复合类查找，供 loader/parser/validate 复用，避免多处重复实现导致支持范围漂移。
- `class_format_parser.py` 负责类格式语法解析，统一输出复合节点描述；在方法级别复用 IR FactoryContext，并在合并子图时对发生冲突的节点/连线执行重命名，确保多方法拓扑不会互相覆盖；**若发生重命名，会同步更新虚拟引脚的 `mapped_ports.node_id`**，避免结构校验因“虚拟引脚仍指向旧 node_id”而误报“缺少数据来源”。同时为 `@flow_entry` / `@event_handler` / `@data_method` 方法建立虚拟引脚与内部端口的映射（流程引脚与数据引脚统一通过 `VirtualPinConfig` 表达）。合并完成后会调用 `engine.graph.semantic.GraphSemanticPass` 覆盖式生成 `signal_bindings/struct_bindings`，避免解析过程多源写入导致的不确定行为。
- `param_usage_tracker.py` 统计复合节点各参数的读取/写入情况，并支持在类格式中识别简单的 `self.xxx` ← 入口形参别名，用于跨方法的虚拟引脚映射；为避免 IR 自动插入的同名节点导致“调用 ↔ 节点”错配，参数使用采集在匹配节点时优先基于源码行号范围（`call.lineno` ∈ `node.source_lineno~source_end_lineno`）定位目标节点。
- 对基于 `if` / `match` 的控制流，类格式解析会识别 IR 中的双分支/多分支节点：
  - 若入口形参仅在条件表达式中使用，则优先将对应的数据输入虚拟引脚映射到分支节点的条件输入端口（`条件` / `控制表达式`），否则退回为 `allow_unmapped` 标记，避免误报“未使用引脚”。
  - 流程类虚拟输出引脚会优先基于源码 AST 定位到对应的控制流分支，并将该虚拟流程出口锚定到生成的【双分支/多分支】节点出口端口（如 `是/否/默认/case值`）。支持常见的稳定写法：
    - 分支体内出现 `流程出("出口名")` 且同级后续存在 `return`（常见于 `流程出 + return` 写法）。
    - 控制语句（if/match）在同级块内“立刻 return”终止，且分支体内 `流程出("出口名")` 位于分支块末尾（常见于入口方法最后统一 `return 数据出...` 的模板写法）。
    - 控制语句（if/match）为方法/语句块的最后一条语句，且分支体内 `流程出("出口名")` 位于分支块末尾（常见于“仅用分支选择流程出口、不额外写 return”的极简模板写法）。
  - 无法静态定位的流程出口回退到方法末尾的终止流程端口；当存在多个分支终止端口时会映射到**所有**终止流程端口，以支持“同一个流程出在多分支汇合后继续连线”的语义。
  - 循环节点（有限循环/列表迭代循环）内部的“无出边流程端口”语义上属于 continue（本次迭代结束），不会被当作方法级流程出口参与虚拟流程出映射。
- 解析算法依赖 `engine.utils.graph.graph_utils` 等图语义工具，保持与核心图模型一致。
- 自动引脚推断：当装饰器省略 `inputs`/`outputs` 时，根据 `pin_marker_collector` 的结果与方法签名补全虚拟引脚，并透传数据出变量映射给 IR 层；同时将所有数据出引脚声明的变量注册为“预声明局部变量”，确保即便首次赋值是函数调用也能直接生成“获取/设置局部变量”组合。流程入虚拟引脚映射时，会优先选择图中无入边的流程节点，若存在【双分支/多分支】则优先把流程入连到它，避免首个流程节点被其它无入边节点抢占。

# 注意事项
- 保持纯逻辑实现，不访问磁盘/网络/UI。
- 避免和 `engine/graph/composite_code_generator.py` 形成循环依赖，必要数据通过参数传入。
- 解析结构需与当前复合节点格式保持一致，新增语法请同步生成与校验流程。
