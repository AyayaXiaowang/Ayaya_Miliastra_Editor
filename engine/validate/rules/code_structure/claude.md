## 目录用途
- 存放“代码结构规范”（M2）相关的规则实现，每个文件聚焦一个主题（if 条件、可变参数最小入参、图变量声明、信号参数名、类型名、常量赋值、事件名、局部变量初始值等）。
- 对外稳定入口仍为 `engine.validate.rules.code_structure_rules`，该文件仅负责 re-export，避免外部 import 路径随内部拆分改变。

## 当前状态
- 已按规则域拆分为多个小模块，降低单文件体积与耦合，便于缓存与性能优化点按域落地。
- 规则实现仅依赖引擎层模块（`engine.*`），不依赖 `app/*` / `plugins/*` / 任何内部开发工具链。
- 已包含“未知节点函数名”校验：当代码出现 `某函数(self.game, ...)` 形态但该函数名不在节点库中，会直接报错，避免拼写错误或不存在的节点名被静默跳过。
- 已包含“已知节点必须传 game”校验：只要函数名在节点库中，就必须显式传入 `self.game/game`（支持位置参数或 `game=self.game` 关键字形式），避免漏传 game 绕过部分静态规则并在运行期报错。
- 已包含【获取局部变量】用法校验：调用结果必须二元拆分赋值或显式下标选择输出（`[0]=句柄`、`[1]=值`），避免误把二元返回当作单值使用。
- 已包含“内置事件回调命名”校验：当 `register_event_handler` 注册的是内置事件时，回调必须为 `on_<事件名>`（严格一致）；信号事件（signal_id/信号名称）不强制回调名。
- 已包含“内置事件回调签名”校验：当 `register_event_handler` 注册内置事件且回调为标准命名 `on_<事件名>` 时，回调参数名集合必须与事件节点输出端口一致（剔除流程端口），避免运行期 kwargs 绑定失败。
- 若事件节点包含 **`泛型` 输出端口**，即便回调方法体内未使用该参数，也必须在代码中显式绑定为具体中文类型（形参注解或占位注解赋值），否则端口类型无法确定。
- 事件名/回调名相关规则支持“模块顶层字符串常量”（含注解赋值）作为事件名来源：例如 `事件名常量 = "定时器触发时"` 后传入 `register_event_handler(事件名常量, ...)` 仍会被校验；避免通过变量间接传入导致规则被跳过。
- 已包含 `on_XXX` 方法名严格校验：只要方法名以 `on_` 开头，`XXX` 必须是内置事件名或已定义信号名/ID；即使没有注册也会报错，防止“写了假的事件入口”长期潜伏。
- client 技能节点图的统一入口 `on_节点图开始` 属于运行时约定（不来自事件节点库），校验规则对该名称做了兼容放行。
- 类型名校验（`type_name.py`）与图变量/结构体相关规则统一以 `engine/type_registry.py` 作为规范中文类型名与别名字典解析的唯一事实来源，并在工作区维度合并节点库端口类型作为补充允许集合，避免多处硬编码类型清单漂移。
  - 额外约束：显式中文类型注解禁止使用“泛型家族”占位类型（`泛型/列表/泛型列表/泛型字典`），包含别名字典 `键类型-值类型字典` 的键/值类型；代码里的显式标注必须收敛为具体类型（泛型仅允许作为端口/编辑期占位存在）。
- 已包含“节点调用必填入参”校验：基于节点库静态输入端口清单，禁止 Graph Code 漏传必填端口参数（流程端口与变参占位端口除外）；对于声明了 `NodeDef.input_defaults` 的输入端口，视为可选入参并允许省略。
- 已包含【设置/获取节点图变量】声明校验：当 `变量名` 可静态解析为字符串时，必须在文件顶部的 `GRAPH_VARIABLES` 中声明；若 `变量名` 来自运行期表达式，则跳过“声明存在性”强校验（降级为 warning）。
- 已包含【获取/设置自定义变量】变量名校验：当 `变量名` 可静态解析为字符串时，必须为非空字符串字面量或模块顶层字符串常量；若来自运行期表达式，则跳过强校验（降级为 warning）。
  - 工程化补充：对 `ui_` 前缀且为 ASCII `snake_case` 的自定义变量名启用长度上限校验（避免写回链路/真源侧截断导致难排查问题）；超长将报错 `CODE_CUSTOM_VAR_NAME_TOO_LONG`。
- 工程化：当 UI源码(HTML) 文本占位符引用到 `ui_*` 自定义变量时，节点图代码中对该变量的【获取/设置自定义变量】必须写到正确的“目标实体”（变量归属）：
  - `lv.` → 归属【关卡实体】
  - `ps./p1..p8.` → 归属【玩家实体】
  - 若代码将 `目标实体` 写为 `self.owner_entity`，则必须在文件头 docstring 显式声明 `mount_entity_type/owner_entity_type: 关卡/玩家` 使其归属明确；否则报错 `CODE_UI_LEVEL_CUSTOM_VAR_TARGET_ENTITY_REQUIRED` 并给出修复建议（声明归属或显式取实体后读写）。
- 已包含“on_实体创建时 冗余图变量初始化”提示（warning）：当【设置节点图变量】把变量设回 `GRAPH_VARIABLES.default_value` 时报告 `CODE_GRAPH_VAR_REDUNDANT_INIT_DEFAULT`，用于提醒删除无意义的默认值重置。
- 已包含“on_实体创建时 冗余自定义变量初始化”提示（warning）：当【设置自定义变量】在 `on_实体创建时` 中写入“可静态识别的常量初始值”（含列表 clear 后写回空列表）时报告 `CODE_CUSTOM_VAR_REDUNDANT_INIT_ON_ENTITY_CREATED`，用于提醒删除无意义的初始化写入并把默认值收敛到实体/模板的变量定义。
- `boolean_conditions.py` 中的布尔条件提示规则会识别 `是否相等(self.game, 输入1=..., 输入2=...)` 这类写法，并给出更直观的 `if 变量:` / 正向条件建议。
- `boolean_conditions.py` 的 if 条件规则已“受控放开”内联表达式：允许在 if 条件中直接书写可归一化的比较/逻辑表达式（例如 `A == B or C != D`），并在校验入口先由 `SyntaxSugarRewriteRule` 改写为等价节点调用（是否相等/逻辑或运算等）；不支持/禁止的比较形态（链式比较、`is/is not`、部分 `in` 写法等）由语法糖改写阶段直接报错（fail-closed），避免 DSL 变成“半 Python”。
- 必填入参与布尔/事件名等规则会按节点图文件推断 scope（server/client），并使用该 scope 下的节点端口定义进行校验，避免同名节点在不同作用域端口不兼容时发生误报。
- 信号/结构体/自定义变量/节点图变量等关键语义节点的识别统一通过 `engine.validate.node_semantics`（语义 ID → 节点 key / alias / #scope 规约），并复用 `engine.graph.common` 中集中定义的端口名常量（如 `VARIABLE_NAME_PORT_NAME`、`STRUCT_NAME_PORT_NAME`）。
- 必填入参/可变参数最小入参规则在统计位置参数时会过滤 `self.game/game/owner_entity` 等保留参数（与 IR 的 `arg_normalizer` 口径一致），兼容节点函数“是否需要显式传 game”的差异。
- 已包含结构体“结构体名”值校验：要求可静态解析且必须匹配结构体定义中的名字（`STRUCT_PAYLOAD.struct_name`），并存在于**当前作用域**（共享 / 共享+当前项目存档）的结构体定义列表中；禁止把 `STRUCT_ID` 当作“结构体名”传入，避免拼写错误/跨项目串包长期潜伏。
- 已增强“发送信号”校验：`信号名` 端口属于静态配置端口，必须为字符串字面量或模块顶层字符串常量；禁止使用运行期变量（包括通过 `数据入` 暴露的引脚值）为其赋值，避免出现 UI/运行期不支持的“连线驱动信号名”形态。
- 工程策略收敛：新增规则 `SharedSignalsForbiddenRule`，禁止在共享根 `assets/资源库/共享/管理配置/信号/` 中放置任何信号定义（`*.py`），避免作用域与 ID 解析不稳定导致导出结果漂移；信号定义必须放在各自项目存档的 `管理配置/信号/` 下。
- 已增强“信号名误用 ID”检测：除完整 `signal_id` 外，也会识别“去掉 `__<package_id>` 后缀的 ID 前缀”写法，并提示改用信号名称。
- 已增加“资源归属边界”校验：当节点图文件位于资源库目录结构下（共享/项目存档），则信号与结构体的**定义来源**必须位于“当前项目存档根目录”或“共享根目录”。若引用到其他项目存档的信号/结构体，将直接报错并指引作者在当前项目目录 `管理配置/信号` / `管理配置/结构体定义` 下新建或补齐定义后再使用；归属推断与路径显示统一通过 `resource_scope_utils.py` 辅助函数实现。
- `literal_assignment.py` 的字面量赋值规则：
  - 方法体内允许使用 **“带中文类型注解 + 字面量赋值”** 作为端口常量的可读写法（例如 `零: "整数" = 0`），避免为了得到 0/1 而引入语义不匹配的节点调用。
  - 模块顶层允许声明“命名常量”（同样形式），用于为节点入参提供可复用常量值；仍禁止在方法体内用赋值语句把“带中文类型注解的字面量常量”（包含模块常量与方法内端口常量）复制到其它变量（`CODE_NO_CONST_ALIAS_ASSIGNMENT`），避免把常量当作“变量初始化来源”产生误导。
- 已包含 ID 字面量格式校验：当变量的类型注解为 `GUID/配置ID/元件ID` 且使用字面量赋值（命名常量）时，值必须为 **1~10 位纯数字**（支持整数或数字字符串）；非纯数字写法会报错 `CODE_ID_LITERAL_DIGITS_1_TO_10_REQUIRED`（普通节点图与复合节点均生效）。
- 已包含 ID 端口常量格式校验：当节点输入端口的期望类型为 `GUID/配置ID/元件ID` 且传入值可静态解析为常量时，必须为 **1~10 位纯数字**（支持整数或数字字符串）；否则报错 `CODE_ID_LITERAL_DIGITS_1_TO_10_REQUIRED`。
- 工程化扩展：当端口期望类型为 `GUID` / `整数`（及其列表类型）时，允许使用字符串占位符 `ui_key:<key>` / `ui:<key>` 作为常量输入；当节点图位于资源库目录结构下且存在 UI源码(HTML) 时，会校验该 key 是否已在 HTML 的 `data-ui-key` / `data-ui-state-group` 中声明（同时兼容 Workbench/写回链路生成的“长 ui_key”，会自动提取其中的关键标识片段），否则直接报错。
- 已包含 `GRAPH_VARIABLES` 中 ID 默认值校验：当 `GraphVariableConfig.variable_type` 为 `GUID/配置ID/元件ID`（及其列表类型）且显式提供 `default_value` 时，要求其可静态解析且满足 **1~10 位纯数字** 约束（列表逐元素校验）；GUID（含 GUID列表）允许 `ui_key/ui:` 占位符，但当节点图位于资源库目录结构下且存在 UI源码(HTML) 时，占位符 key 必须在 `data-ui-key` / `data-ui-state-group` 中声明，否则报错。
- 已包含 `GRAPH_VARIABLES` 中“整数/整数列表”默认值校验：当 `GraphVariableConfig.variable_type` 为 `整数/整数列表` 且显式提供 `default_value` 时，要求其可静态解析且为数字（或 `ui_key:<key>` / `ui:<key>` 占位符；列表逐元素校验）；当节点图位于资源库目录结构下且存在 UI源码(HTML) 时，占位符 key 必须在 `data-ui-key` / `data-ui-state-group` 中声明，否则报错。
- 已包含结构体图变量默认值约束：当 `GraphVariableConfig.variable_type` 为 `结构体/结构体列表` 且提供了**非空 default_value** 时，必须同时提供 `struct_name` 且指向已存在的结构体定义（否则报错），用于保证后续写回存档/编辑器解析时结构体类型可对齐。

## 注意事项
- 新增规则时请放入独立模块，并在 `__init__.py` 与 `engine.validate.rules.code_structure_rules` 的 re-export 列表中补齐。
- 避免跨模块互相 import 形成环；共享逻辑优先抽到同目录的独立 helper 模块（如后续确实需要）。
