## 目录用途
校验规则实现与规则侧工具：将 `engine.configs.rules.*` 的权威规则/类型体系，结合节点库与 AST/Schema 视图，转化为可组合的 `ValidationRule` 并在校验管线中执行。

## 当前状态
- 以“纯规则 + 轻量工具函数”为主，为 `engine.validate.*` 提供可组合规则集；读取源码默认使用 `utf-8-sig` 兼容 Windows 常见 BOM。
- 规则会按节点图文件路径推断 graph scope（server/client），选择对应作用域的节点库视图参与端口名/端口类型/必填入参等校验，避免跨域覆盖导致误报与缓存不稳定。
- GraphModel 解析优先走“尽力解析 + 缓存复用”路径；IR 建模错误（`GraphModel.metadata["ir_errors"]`）与图结构错误（`validate_graph_model`）在规则侧提升为 error，以对齐严格加载语义。
- 命名约束：自定义变量名与节点图变量名长度上限 **20 字符**，在规则层强制为 error（便于在源码阶段尽早拦截）。
- 语法糖归一化（列表/字典/比较/下标/and/or 等）在校验入口执行 rewrite；rewrite 的配置与上限以 `engine/graph/utils/graph_code_rewrite_config.py` 为单一真源。
- 复合节点相关规则以真实语义来源为权威：payload 读取 `COMPOSITE_PAYLOAD_JSON`，类格式读取 pin_marker 扫描结果；不再兼容旧函数式复合节点格式。
- 复合节点校验额外包含“解析子图后的结构与连通性”规则：
  - 子图结构校验：对子图执行 `validate_graph_model`（对齐 UI 严格加载），避免“validate-file 通过但 UI 报错”。
  - 虚拟引脚映射校验：禁止存在 `mapped_ports=0` 且 `allow_unmapped=False` 的悬空虚拟引脚（避免 UI/导出/写回侧出现“引脚缺失/空白”）。
  - 流程连通性校验：当同时存在流程入/流程出虚拟引脚时，流程出必须从任一流程入可达（支持分支/循环）。
- 端口类型校验覆盖“节点函数调用”与“复合节点实例方法调用”（`self.<alias>.<method>(...)`）；复合调用的端口类型以节点库中的复合 `NodeDef.input_types` 为准，避免复合节点入参类型错误漏报。
- 复合节点源码同样执行端口类型与同型输入约束（含 `拼装列表/拼装字典` 的同型检查），避免“同样写法在节点图能报错、在复合节点内不报错”的口径漂移。
- 对 `三维向量列表/配置ID列表/元件ID列表/阵营列表` 等强类型列表端口：允许直接写列表字面量（通常会被归一化为 `拼装列表`），端口类型匹配以“连接目标端口类型”为准；UI/写回侧依赖 `GraphModel.metadata["port_type_overrides"]` 获取该列表的有效类型，避免被误判为整数列表/泛型列表。
- 对 `三维向量`：支持在调用入参位置直接写 `(x, y, z)`（括号 tuple）作为端口常量；校验会校验其形态与数值字面量约束，避免预览中被误判为整数列表或引入多余节点。
- 对 `三维向量` 的 tuple 常量：允许引用**模块级数值常量**（如 `VEC_X = 1.0` 后写 `(VEC_X, 2.0, 3.0)`），以对齐“常量复用”写法与字面量等价的直觉口径。
- 对 `三维向量列表`：允许列表字面量/`拼装列表`，但**列表元素必须是三维向量值**（`(x, y, z)`、`创建三维向量(...)`、或类型为『三维向量』的变量）；禁止元素写成 `[x, y, z]` 这类“列表形式的向量”。
- 对除 `三维向量列表` 以外的**具体列表类型端口/注解变量**（如 `浮点数列表/整数列表/字符串列表`）：当入参/赋值右侧可静态提取为列表元素（列表字面量或 `拼装列表(...)`）时，校验会逐元素检查类型必须匹配对应 `base_type`，避免 `泛型列表` 连线规则放行导致的元素类型错误漏报。
- 复合节点校验不再阻断“复合内嵌套复合”；嵌套复合的可否用解析/写回链路的真实能力为准（失败直接抛错暴露问题）。
- UIKey 占位符存在性校验以 UI源码(HTML) 为真源：
  - 普通控件：`data-ui-key`
  - 多状态：`data-ui-state-group` + `data-ui-state`（支持校验 `UI_STATE_GROUP__<group>__<state>__group` 的 state 是否真实存在）
  - `component/entity` 占位符在校验阶段仅做非空语法校验（存在性由导出/写回阶段决定）。
- 类型体系（类型名/别名/转换/容器约束）唯一事实来源为 `engine/type_registry.py`；数字 ID（1~10 位纯数字）判定复用 `engine.utils.id_digits.is_digits_1_to_10`。
- 节点挂载/数据类型等配置真源在 `engine.configs.rules.*`；本目录仅消费其视图或提供兼容 re-export，避免维护平行清单。
- `node_index.py` / `ast_utils.py` / `node_semantics` 提供节点库速查、AST 工具与语义节点定位入口；规则应使用语义 ID 表达意图，避免硬编码节点标题字符串。

## 注意事项
- 不在本层维护平行“类型清单/规则常量”；新增规则优先复用节点库视图、Schema 视图与现有 AST 工具。
- 规则实现不使用 try/except 吞错；strict vs best-effort 的策略由 orchestrator/调用方决定。
- 文档字符串与注释中不要写外部系统的具体路径，保持抽象描述层级。
