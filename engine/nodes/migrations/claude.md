## 目录用途
存放“节点库变更的迁移规则模型”（breaking 变更的显式兼容/自动迁移支撑）。

本目录只提供**纯数据结构与纯函数**，用于描述：
- 节点改名（Graph Code 调用名替换）
- 端口改名（关键字参数名替换）
- 枚举值迁移（枚举端口候选值替换）
- （预留）复合节点方法名迁移等

## 当前状态
- 目前提供 `node_migrations.py`：定义迁移规则的数据模型与默认迁移计划入口 `get_default_migration_plan()`。
- 默认迁移计划保持为空：优先依赖 manifest diff 的“按 origin/composite_id 识别改名”能力；当出现无法自动推断的端口改名等 breaking 变化时，再在此显式登记迁移规则。

## 注意事项
- 禁止在本目录做 I/O（不读写文件、不访问 UI、不访问资源库）；迁移工具属于内部工具链，不随仓库公开分发。
- 规则命名应稳定、可审阅；涉及节点身份时优先使用 `semantic_id` 或 manifest 的 `origin/composite_id` 作为锚点，避免依赖显示名字符串漂移。
