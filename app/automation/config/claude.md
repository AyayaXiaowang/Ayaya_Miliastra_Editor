## 目录用途
- 自动化执行器的“节点配置流程”实现：参数注入、Settings 行定位、分支/变参端口新增、信号绑定等。
- 与 `app.automation.editor` 协作：本目录只做**流程编排与可复用小步骤**，不直接依赖 UI 组件。

## 当前状态
- **参数配置**：`config_params.py` / `config_node_steps.py` 将“配置一个节点”拆成可测试的小步骤（端口定位、端口间距法输入框定位、Warning 搜索区域计算、布尔/枚举/三维向量输入等）；为避免“输入过程中每帧识别漂移”导致端口候选不稳定，参数配置会固定使用**首帧端口快照**进行端口定位与上下文判断，并在后续参数循环中仅抓取“当前窗口截图”用于可视化与模板匹配，不再每个参数都重新定位节点 bbox/重新识别端口，从而去掉多余的 1~2s 停顿。普通参数默认走**端口间距法**（以端口中心点为基准）：若当前端口与下一端口/节点底边的垂直间距 ≥ 1 个端口高度，则点击点为 `(+2*port_w, +1*port_h)`；否则点击点为 `(+2*port_w, +0*port_h)`，不再依赖 `Warning.png` 模板，也不做端口偏移回退点击；`Warning` 搜索区域仅在**三维向量几何法**与**枚举/布尔的 Warning 展开重试**时按需计算。对输入端口名为“变量名”的常量值会做一次轻量归一化：当检测到旧格式 `var_xxx`（关卡变量 ID）时，优先查 `LevelVariableSchemaView` 映射为中文 `variable_name` 再注入编辑器，避免执行步骤引导用户填写难读的英文 ID（对“节点图变量/局部变量”节点会自动排除）。
- **枚举/布尔下拉**：`handle_enum_param` 已从“固定间距按第 N 项点击”升级为“识别 D7D7D7 下拉矩形 → OCR 选项文本 → 点击目标文本”；当选项过多无法完全展示时，会将鼠标移动到下拉区域内并按“滚动两格→截图→OCR”的节奏分页识别，自动检测滚动到头；当 OCR 部分缺失时会利用枚举固定顺序基于已识别项进行位置推断点击。
  - 纯逻辑辅助集中在 `enum_dropdown_utils.py`（文本归一化 / 缺字顺序推断），便于单元测试回归；文本归一化会同时移除空白与 `_ / ＿`，用于匹配“枚举定义含下划线但 UI 展示/ OCR 结果不含下划线”的情况。
  - 当 OCR 文本与枚举定义无法匹配但“识别条目数 == 枚举总数”时，会启用**顺序兜底**：仅在第一页（未滚动）按从上到下顺序一一映射并点击目标序号（避免滚动分页场景误判）。
- **分支配置**：`branch_config.py` 处理多分支类节点的端口新增与分支输出配置，复用参数配置阶段的端口定位与 Warning 处理。
- **信号绑定**：`signal_config.py` 提供“设置信号”步骤，优先通过 **OCR 在节点内定位【信号名】文本**（顶部区域排除优先复用“一步式识别”的节点标题栏高度动态结果，必要时回退到 profile 静态高度）后再按几何关系点击下方输入行注入；当 OCR 未命中时回退到 `Signal.png` 模板匹配作为兜底，避免偶发漏检直接失败。
- **结构体绑定**：`struct_config.py` 提供“配置结构体”步骤，定位结构体节点后优先在端口识别结果中找到【结构体名】行并点击其 Settings 图标；若无法定位该行则回退为“节点内最上方的 Settings 模板命中”。随后复用“类型搜索框”的交互（搜索框模板 + 输入 + 点击候选）选择结构体名称，最后点击画布空白位置收尾并标记节点快照为脏。
- **快照缓存**：参数与端口相关步骤统一复用 `app.automation.editor.node_snapshot.NodePortsSnapshotCache`，尽量共享同帧截图与端口列表以降低 OCR 频率。
  - `settings_scanner.py` 仅负责收集 Settings 行的位置（side/index/center/bbox 等）并做可视化记录；端口名来源统一以快照中已填充的 `name_cn` 为准（由执行器解析到唯一 `node_def` 后映射得到），不在配置链路中再做“按节点中文名全库反查”的映射。

## 注意事项
- 所有截图/模板匹配/端口识别必须经 `app.automation.capture` 与 `app.automation.vision`，不要在本目录直接访问底层实现。
- 不新增 `try/except` 吞错；失败应通过返回值与日志显式暴露，由上层决定是否重试或中止。
- 仅通过 `EditorExecutorWithViewport` 等协议访问执行器能力，禁止跨模块调用私有方法或依赖具体 UI 实现细节。
- 本目录只承载“配置流程”逻辑；其它能力按职责放在 `app.automation.editor/`、`capture/`、`vision/`、`ports/` 子包中。
