## 目录用途
执行监控面板（Execution Monitor）的 UI 侧实现：在主窗口中展示截图/识别叠加、日志与事件列表，并提供执行控制（执行/暂停/单步/终止）与少量识别/定位测试入口。

## 当前状态
- 面板本体以“胶水层”为主：负责控件组装与依赖注入，将截图抓取、可视化渲染、日志展示、识别动作、定位控制、执行控制等逻辑分散在子模块中维护。
- 与 UI 无关的纯策略已下沉到运行时 service（例如精简模式的尺寸/布局推导），本子包以 UI 适配与交互为主。

## 注意事项
- UI 线程安全：跨线程更新必须通过 Qt 信号回到 UI 线程，避免直接操作控件。
- OCR/识别依赖缺失时允许降级为只读浏览模式（可启动/可看任务清单，但禁用执行与识别测试）。
- overlays/叠加层结构保持轻量（dict/list），避免在此子包引入新的数据模型层导致连锁依赖。
- 对外导入入口保持稳定：优先 `from app.ui.execution.monitor import ExecutionMonitorPanel`（父包如有再导出也需同步维护）。
