## 目录用途
`app/ui/dialogs/` 存放各类对话框组件与可复用的“对话框/面板内嵌编辑器”，用于配置编辑、信息查看与用户确认等交互。

## 当前状态
- **统一骨架**：大多数对话框继承 `BaseDialog` 或管理类对话框基类，由基类提供标题、内容区、标准按钮区与主题样式；业务对话框只负责组装表单/表格并导出结果。
- **全局设置对话框**（`settings_dialog.py` / `SettingsDialog`）：集中呈现并写回 `engine.configs.settings.settings` 的全局开关；采用左侧分类导航（性能/排版/画布/任务清单/输出与调试/执行与系统），左侧分类列表复用 `ThemeManager.left_panel_style()`（`objectName="leftPanel"`）统一选中态与交互风格，避免在对话框内写局部 QSS 造成样式分叉。其中“性能”集中收敛画布性能相关项：画布 LOD（分层绘制：节点细节/标题/端口/连线阈值、网格低倍率稀疏、blocks-only 鸟瞰进入/退出阈值与鸟瞰网格间距）、行内常量控件虚拟化、超大图快速预览开关与触发阈值、自动适配全图；“排版”包含自动排版行为开关与节点间距倍率；“画布”包含节点内容区不透明度与 basic blocks 背景显示/透明度；其余分类包含任务清单、调试输出与执行/系统参数，并在“输出与调试/输出与打印”中提供 `UGC_GIA_NODE_POS_SCALE`（节点图 `.gia` 导出节点坐标缩放倍数）。点击“确定”后会立即读取当前 `GraphView.transform().m11()`（真实缩放比例）并调用 `GraphScene.set_view_scale_hint(...)` 重新同步 LOD/可见性与叠加层，确保画布无需再次滚轮缩放也能立刻按当前倍率应用新阈值。**本地增强功能**与**资源库自动更新**已改为始终启用，不再在设置页展示。
  - 画布网格：新增“显示画布网格”开关（`settings.GRAPH_GRID_ENABLED`），关闭后背景仅保留纯底色不绘制网格线，可显著降低超大图平移/缩放时的背景绘制开销；网格参数（最小像素间距）会随开关禁用。该开关与 LOD 总开关解耦（不依赖 `GRAPH_LOD_ENABLED`）。
  - 拖拽平移静态快照：新增“拖拽平移期间冻结为静态快照”开关（`settings.GRAPH_PAN_FREEZE_VIEWPORT_ENABLED`），在手抓拖拽平移画布期间抓取 viewport 快照并冻结画面以避免每帧重绘大量 items，松手后恢复真实渲染（代价是拖拽平移中不显示新进入视口的内容）。
  - 滚轮缩放静态快照：新增“滚轮缩放期间冻结为静态快照（极致性能）”开关（`settings.GRAPH_ZOOM_FREEZE_VIEWPORT_ENABLED`），滚轮缩放期间抓取 viewport 快照并冻结画面以避免每步滚轮重绘大量 items，停止滚轮后恢复真实渲染（代价是缩放中不显示新进入视口的内容）。
  - 平移/缩放隐藏图标：新增“平移/缩放期间隐藏端口/图标”开关（`settings.GRAPH_PAN_HIDE_ICONS_ENABLED`），平移（拖拽）或滚轮缩放期间临时隐藏端口/⚙/+ 等小图元并跳过 YDebug 调试叠层绘制，停止交互后恢复，用于减少 Qt item 枚举与绘制固定开销。
- 设置页“性能”中包含**画布性能面板**开关：开启后在画布左上角显示拖拽/缩放/重绘的耗时分解，并提示最大开销段，用于定位超大图平移卡顿来源；默认关闭以避免日常使用的额外统计成本。
- 设置页“性能”中包含**全局性能监控（卡顿定位）**：用于定位“非画布页面/全局 UI 线程卡顿”。开启后会记录 UI 主线程阻塞事件，并可通过“性能面板”查看卡顿时刻的主线程调用栈；可选显示主窗口级悬浮面板（全页面可见，点击打开详情面板）。
- **性能监控面板**（`performance_monitor_dialog.py`）：非模态对话框，展示全局性能监控报告（UI gap/卡顿事件/命名耗时段）。为避免“监控面板自身高频刷新大段文本造成卡顿”，对话框会将实时指标（gap/avg/max、阈值、事件计数）放在顶部状态行高频刷新，而“卡顿/耗时段详情文本”仅在事件变化时刷新；支持“**一键复制**”（复制时附带实时指标）与清空记录（仅清空已记录事件，不影响后续采集；必要时会触发监控自愈以避免 watchdog 异常退出后不再产出事件）。
- **结构体定义编辑**：结构体字段编辑遵循“显式点击后再编辑”，集合类型（列表/字典）以内联子表格展开；只读场景作为浏览视图使用，增删改在代码资源中维护；当上层进入只读/置灰状态时，列表/字典的子表格也会同步灰显并完全禁用编辑与删除（仅保留展开/折叠与滚动浏览），避免出现“看似可改但无法保存”的交互错觉；**支持别名字典类型**（`键类型-值类型字典` / `键类型_值类型字典`），在 UI 中按“字典”控件编辑，并在导出默认值节点时据别名写入对应的 `key_type` / `value_type`（内部字段使用英文 `String/Int32/...` 以与既有资源保持一致）。
- **结构体列表条目查看/编辑**（`struct_list_item_dialog.py`）：提供结构体列表条目的字段编辑对话框（按结构体定义展示字段并写回 `{字段名: 值}`），以及只读查看对话框用于在外部变量等只读场景下浏览条目原始 fields（JSON 展示，便于复制与排查）。
  - 结构体定义与结构体 ID 下拉选项应优先使用上层注入的 `resource_manager` 作用域（共享根 + 当前存档根），避免在共享视图混入其它项目存档目录下的结构体定义。
- **信号编辑器**（`signal_edit_dialog.py` / `SignalEditorWidget`）：
  - 参数表格使用“工具栏 + 表格”的内联模板展示“参数名 / 数据类型 / 描述”，并通过表格右键菜单提供行级操作。
  - 只读模式下禁用参数增删与单元格编辑控件，并隐藏清除（X）按钮；单元格容器会铺满整格的禁用底色以避免出现“置灰未填满”的视觉空洞；点击只读内容会弹出轻提示：**当前内容暂不支持在UI中编辑。**
- **变量编辑对话框**（`variable_edit_dialogs.py`）：仅保留节点图变量的 `GraphVariableEditDialog`；实体/模板自定义变量已迁移为关卡变量代码定义，不再提供“新建/编辑实体变量”的弹窗入口。
- **本地测试对话框**（`local_graph_sim_dialog.py`）：在主程序内启动“节点图 + HTML UI”的本地模拟（HTTP server + 浏览器预览），用于验证信号/交互/显隐与状态切换的最短闭环；不依赖 WebEngine（默认走系统浏览器）。
  - 支持配置在场玩家数量（用于“获取在场玩家实体列表/等待其他玩家/投票门槛”等多人逻辑模拟）。
  - 支持配置本地 HTTP 监听端口（基准端口）：默认 `17890`（可用 `AYAYA_LOCAL_HTTP_PORT` 覆盖；兼容 `AYAYA_LOCAL_SIM_PORT` 别名）；占用则向上顺延扫描，扫描不到回退为系统临时端口；设为 0 则强制使用系统临时端口。
  - 对话框内容区顶部保持简洁：不额外展示“用途/当前项目路径”等说明性文本。
  - 节点图选择以**当前项目存档目录**为作用域：左侧按文件夹浏览（目录树），右侧总览列表勾选运行/设为入口；勾选的节点图会统一挂载到对话框右侧的 `owner` 实体名。若需要“挂到其它实体”，应通过“元件/实体挂载”扫描与勾选完成。
  - 入口图 `owner` 支持按“引用资源”自动推断：从入口图源码读取 `graph_id`，查询当前项目存档内引用该图的元件模板/实体摆放/关卡实体；若能唯一确定则自动填充 `owner`，若存在多条引用则提供下拉选择，并可通过“刷新引用”确保与资源库变更同步。
  - UI HTML 选择同样限制在当前项目存档的 `管理配置/UI源码/` 下，避免在全文件系统中误选其它项目文件。
  - 对话框复用/再次打开时会同步当前项目存档作用域并刷新可选列表，确保始终只看到“当前项目”文件。
  - 支持“元件/实体挂载”扫描与勾选运行：从当前项目存档扫描模板/实例/关卡实体，展示“哪个元件/实体挂了哪些节点图”，并可勾选将其自定义变量默认值预加载到模拟实体上（变量文件默认值 + 自定义变量组件默认值 + 实例覆写；实例可选同时挂载模板 `default_graphs`）。

## 注意事项
- 尽量复用 `ThemeManager` 的样式工厂与通用表格/表单骨架，避免在对话框内硬编码颜色与交互细节。
- 自定义控件背景/边框颜色时，优先使用 `ThemeManager/Colors` 已存在的 token（例如 `Colors.BG_CARD/BORDER_LIGHT`），避免引用不存在的旧 token 导致启动期直接崩溃。
- 本目录避免使用 `try/except` 兜底；错误直接抛出由上层统一处理。
