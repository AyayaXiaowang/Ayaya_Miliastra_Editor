## 目录用途
主窗口控制器层：将“业务编排/资源生命周期/导航与保存策略”等从具体 UI 组件中拆出，通过信号槽与主窗口、各页面协作，降低耦合并便于测试。

## 当前状态
- 覆盖的核心领域包括：项目存档生命周期（创建/加载/保存/导入导出）、节点图会话管理（加载/保存/校验/自动排版编排）、跨页面导航协调、文件监控与资源库刷新协作。
- 复杂流程按领域下沉到子目录实现（例如保存链条、节点图加载/排版/保存流程、文件监控与自动刷新桥接），控制器更偏向依赖注入与信号转发。
- UI 源码页面的转换/派生与 Web 入口打开由 `ui_pages_browser.py` 编排；对外入口命名为“UI预览（Web）”，保持与导航按钮一致。

## 注意事项
- 控制器不直接操作具体 Widget：UI 变更通过信号委托给主窗口/页面，避免形成“互相持有引用”的网状依赖。
- 错误策略 fail-fast：不写 try/except 吞错，异常向上传播交由统一异常钩子处理。
- 依赖方向保持单向：controllers 内避免反向依赖具体视图实现细节，防止循环依赖与重复导入。
- 对话框/提示统一走 UI 层封装（如标准确认/输入对话框），不要在控制器里直接 new `QMessageBox`。
- 高频入口（如文件监控回调）优先用纯字符串路径归一化，避免 `Path.resolve()` 等可能触发文件系统 IO 的操作压垮事件循环。
