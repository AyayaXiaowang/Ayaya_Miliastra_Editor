## 目录用途
`ui/graph/logic/` 存放节点图编辑的纯逻辑层代码（无 PyQt 依赖），聚焦可测试的核心决策与数据处理（如信号/结构体节点的端口规划与 NodeDef 代理生成）。

## 当前状态
- 提供信号节点与结构体节点的绑定解析、端口同步规划与 NodeDef 代理构建逻辑，向 UI 层输出可执行计划（新增端口、常量更新、绑定 ID 等）。
- 结构体节点将唯一结构体端口（端口名 `结构体`）实例化为具体类型 `结构体<struct_name>` 用于展示与连线校验；`结构体名` 仅作为 `input_constants` 兼容展示，不再作为真实端口出现。
- 输出仅操作引擎侧数据模型与 NodeDef，不触碰 QGraphicsItem 或对话框，便于单元测试覆盖。

## 注意事项
- 保持模块纯净：禁止引入 PyQt；函数入参仅接受模型/配置数据。
- 逻辑函数应返回明确的计划结果数据结构，由上层决定如何落地到 UI。
- 新增能力优先以“小型纯函数 + dataclass 结果”形式扩展，确保可复用与可单测。
