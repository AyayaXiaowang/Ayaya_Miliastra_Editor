目录用途：存放 GraphView 自动排版流程代码（验证、布局、模型差分合并、图形项同步）。
当前状态：AutoLayoutController 负责按钮入口，并将“验证 + LayoutService.compute_layout”放到后台线程 `_AutoLayoutComputeThread` 执行以避免阻塞 UI（默认展示 GraphView 的加载遮罩，阻断交互避免线程期间模型被改动）；线程返回验证错误时，Toast 仅提示首条错误并提示在“设置>自动排版”开启“图编辑器详细日志”查看详情，若 settings.GRAPH_UI_VERBOSE 为 True 同时在控制台打印完整错误；验证规则：普通节点图优先解析 `metadata.source_file` 并走 `engine.validate.validate_files`，复合节点编辑器（存在虚拟引脚映射）走 `engine.graph.validate_graph_model`；布局差分合并仍统一复用引擎侧 `engine.layout.utils.augmented_layout_merge.apply_augmented_layout_merge` 保持“单一真源”；合并后 UI 侧使用主线程分帧任务 `_AutoLayoutApplyJob`（QTimer time-budget）清空并重建图元（期间设置 `scene.is_bulk_adding_items=True` 并在结束前 `flush_deferred_port_layouts()`，避免逐边触发端口重排）：删除旧连线图元并清空邻接索引、同步清理 fast_preview_mode 的 batched edges（基于合并前旧 edge_id 列表移除，避免残留绘制）、剔除 Qt 已释放的悬空 node_items、删除 removed_copy_node_ids 的副本节点、补齐新增节点、同步节点坐标、重建连线；最后失效 basic blocks 背景矩形缓存并刷新 `sceneRect`/小地图缓存（优先 `scene.rebuild_scene_rect_and_minimap()`，否则回退 `_update_scene_rect + mini_map.reset_cached_rect`）以避免“平移边界未扩展/小地图拖不动”；当启用“长连线局部变量中转”时，会在合并后提示本次插入的 relay 节点数量（有插入则 Toast；仅在 GRAPH_UI_VERBOSE 下打印 0 插入信息）。
注意事项：调用需传入 GraphView 场景；排版前回调 `view.on_before_auto_layout()` 约定可返回 True 表示已触发“重解析/重载”，AutoLayoutController 会跳过本次排版并等待加载完成后再触发排版；复合节点自动排版依赖 composite_edit_context 提供 manager 与 composite_id 以构建虚拟引脚映射；清理旧图元（连线/节点）与同步坐标前必须使用 `sip.isdeleted()` 剔除 Qt 已释放对象的悬空引用，避免 `wrapped C/C++ object ... has been deleted` 崩溃。
