## 目录用途
- 承载任务清单右侧“详情面板”的 UI 与轻量逻辑：结构化文档模型、builder registry、渲染与交互（折叠块/表格等）。
- 详情构建保持无 Qt 依赖：builder 输出结构化文档，由 View 层负责渲染为 Widget。

## 当前状态
- 详情构建逻辑通过 registry 分发到 `builders/` 下的插件模块（按 detail_type/前缀注册），避免中心化 if/elif。
- 详情面板与执行相关按钮区保持分离：滚动区域承载文档内容，控制按钮固定可见。

## 注意事项
- 不使用 `try/except` 吞异常；构建失败直接抛出，方便定位。
- 避免在模块顶层导入重型依赖；需要时按需/延迟导入，配合 registry 的按需加载策略降低 UI import 卡顿。
