## 目录用途
- 任务清单（Todo）页面的 UI 子系统：左侧步骤树、右侧详情与节点图预览、执行入口桥接、运行态状态展示与导航。
- 仅负责呈现与交互编排；步骤生成/规则判定/执行规划尽量下沉到无 Qt 依赖的服务层或 `app.models`。

## 当前状态
- **页面宿主**：`todo_list_widget.py` 负责布局装配与对外接口，复杂流程由编排/控制器类协助完成（避免宿主膨胀）。
- **列表/编排**：`list/` 收敛“加载/刷新/对外聚焦/快捷键”等页面级编排逻辑（详见 `list/claude.md`）。
- **步骤树**：`tree/` 负责树构建、懒加载、三态/样式与图步骤联动（详见 `tree/claude.md`）。
- **详情面板**：`detail/` 以结构化文档模型渲染详情内容，并通过 builders 拆分不同 `detail_type` 的构建逻辑（详见 `detail/claude.md` 与 `detail/builders/claude.md`）。
- **预览**：`preview/` 管理只读节点图预览、高亮与聚焦（详见 `preview/claude.md`）。
- **运行态**：`runtime/` 收敛执行过程中的运行态状态（完成/失败/跳过等）与 UI 同步策略（详见 `runtime/claude.md`）。
- **协议层**：`ports/` 定义与主窗口/执行监控/执行器等协作的最小协议（详见 `ports/claude.md`），避免跨模块直连具体实现类。
- **杂项**：`misc/` 放置少量与上述主链路解耦的小工具与辅助件（详见 `misc/claude.md`）。

## 注意事项
- **依赖边界**：保持 `app/ui -> app/models` 单向依赖；不要在本目录引入自动化底层实现细节。
- **性能**：UI 线程避免同步磁盘加载/大图解析；需要解析/扫描时应后台化，并在 UI 侧只消费“轻量结果/缓存命中”。
- **稳定依赖入口**：从主窗口获取 workspace/节点库/ResourceManager 等稳定依赖时，统一经 `main_window.app_state` 与公开控制器接口访问，避免依赖旧式兼容别名。
- **执行监控协作**：执行监控面板的显隐与访问统一走主窗口 `right_panel` 的公开 API（tab_id=`execution_monitor`），任务清单不直接操作右侧标签页容器。
- **错误可见**：不做静默兜底；前置条件缺失或协议不匹配应在 UI/日志中明确暴露，便于定位。
